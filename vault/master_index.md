# Master Index — dygyn-bet Vault

## Overview

`vault/` is the project memory system and canonical source of truth for `dygyn-bet`.

It tracks:

- product purpose and legal boundaries
- current architecture
- data model
- API contracts
- code map
- sprint priorities
- exact resume point
- changelog and verification history

## Start Here

At the start of every session, read in order:

1. `vault/master_index.md`
2. `vault/WORKFLOW.md`
3. `vault/sprint.md`
4. `vault/resume-plan.md`

Then read task-specific docs:

- `vault/CODE_MAP.md`
- `vault/wiki/architecture/`
- `vault/wiki/services/`
